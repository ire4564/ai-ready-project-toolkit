# 10 — Philosophy

## Core Beliefs

### 1. 읽기 쉬운 코드가 좋은 코드다

코드는 쓰는 시간보다 읽는 시간이 10배 많다.
"나중에 내가 봐도 바로 이해되는가?"가 가장 중요한 기준이다.

### 2. 작게 만들고, 자주 배포한다

큰 PR은 리뷰하기 어렵고, 버그를 숨긴다.
하나의 PR은 하나의 관심사만 다룬다.

### 3. 타입은 문서다

TypeScript의 타입 시스템은 가장 신뢰할 수 있는 문서다.
`any`는 "나는 이걸 이해하지 못했다"의 다른 표현이다.

### 4. 자동화할 수 있으면 자동화한다

lint, format, test, deploy — 사람이 기억해야 하는 건 실패한다.
CI/CD와 Git hook으로 강제한다.

### 5. 완벽보다 완성

프로젝트의 적은 완벽주의다.
80%의 완성도로 배포하고, 피드백을 받아 개선한다.

## Anti-Beliefs (내가 경계하는 것)

- **문제 주도 개발** — 기술이 멋져서가 아니라, 문제를 풀기에 적합해서 선택한다.
- **과도한 추상화** — DRY를 맹신하지 않는다. 약간의 중복이 잘못된 추상화보다 낫다.
- **유행 따라가기** — 새 라이브러리가 나왔다고 바꾸지 않는다. 현재 도구가 문제를 못 풀 때만 바꾼다.
